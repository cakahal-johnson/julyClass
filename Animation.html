Css||Animations

An Animation lets the HTML elements to gradually change from one style to another which include changing most Css properties and also as many times as you want

Rules:

  1. for any element to Animate we need to hold it in @keyframes which specify the change from time to time by also binding the animation to an element

Properties for Animations:

  1. animation: which a shortcut of all other properties
  2. animation-name: which is used to bind the keyframes
  3. animation-duration: which is time to take
  4. animation-delay: which is time to delay for the change to take place
  5. animation-iteration-count: means loop
  6. animation-fill-mode: means adding a fill to property
  7. animation-direction: this specifies the forward, backwards or alternate cycles
    8. normal: by default forward direction
    reverse: backwards direction
    alternate: starts with forward then backwards direction
    alternate-reverse: starts with backwards then forward direction
  8. animation-timing-function: means loop time speed curve
    9. ease: means take a slow start, then fast, then end slowly (is default)
    linear:
    ease-in:
    ease-out:
    ease-in-out:
    cubic-bezier(n, n, n, n): it define your own values in a cubic-bezier function

Example:

fill-mode

this specifies a style for the target element when the animation is not playing (before it starts, after it ends, or both )

none: means default
forward: means the element will retain the style values that is by the LAST keyframe then it all depends on animation direction and iteration-count
backwards: means the element will GET the style values that is by the FIRST keyframe then it all depends on animation direction and iteration-count IT WILL RESTAIN during the animation-delay period
both: the rules is that the animation will follow both the forward and backwards, extending the animation properties in both directions
